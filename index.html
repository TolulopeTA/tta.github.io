HTML: Forms

Tags

S/N Tag(s) used Name Description
1 <form> form allow users to input information and send it
2 <input> input To add fields for users to input information we use the <input> element and set the type attribute to a field
3 <label> label For a user to properly identify an <input> we use the appropriately named <label> element.
The <label> element has an opening and closing tag and displays text that is written between the opening and closing tags.
4 <select>, <option> select A <select> element is populated with <option> elements and renders a dropdown list selection.
5 <datalist>, <option> datalist A <datalist> element is populated with <option> elements and works with an <input> to search through choices.
6 <textarea> textarea A <textarea> element is a text input field that has a customizable area.


Type Values

S/N Fields Description
1 "text" creates a single row field for text input

Setting type to "text" and adding the list attribute will pair the <input> with a <datalist> element if the list of <input> and the id of <datalist> are the same.
2 "password" creates a single row field that censors text input
3 "number" creates a single row field for number inputs
4 "range" creates a slider to select from a range of numbers
5 "checkbox" creates a single checkbox that can be paired with other checkboxes
6 "radio" creates a radio button that can be paired with other radio buttons
7 "submit" creates a submit button


Attributes

S/N Attributes Used for Description
1 action form The form‘s action attribute determines where the form’s information goes
2 method form The form‘s method attribute determines how the information is sent and processed.
The method attribute is assigned a HTTP verb that is included in the HTTP request.
3 id input To associate a <label> and an <input>, the <input> needs an id attribute
4 name input It’s important that we include a name attribute for the <input> — without the name attribute, information in the <input> won’t be sent when the <form> is submitted.
When a <form> is submitted, the name of the fields that accept input and the value of those fields are sent as name=value pairs.
5 type input To add fields for users to input information we use the <input> element and set the type attribute to a field.
6 value input When a <form> is submitted, the name of the fields that accept input and the value of those fields are sent as name=value pairs.
7 placeholder input A placeholder shows up when the field has no value, it disappears after the user types or selects something in the field.
8 for label To associate a <label> and an <input>, the <input> needs an id attribute. We then assign the for attribute of the <label> element with the value of the id attribute of <input>.


Form Validation

S/N Attributes Description
1 required Adding the required attribute to an input related element will validate that the input field has information in it.
2 min Assigning a value to the min attribute of a number input element will validate an acceptable minimum value.
3 max Assigning a value to the max attribute of a number input element will validate an acceptable maximum value.
4 minlength Assigning a value to the minlength attribute of a text input element will validate an acceptable minimum number of characters.
5 maxlength Assigning a value to the maxlength attribute of a text input element will validate an acceptable maximum number of characters.
6 pattern Assigning a regex to pattern matches the input to the provided regular expression, or regex.
Let’s say we wanted to check for a valid credit card number (a 14 to 16 digit number). We could use the regex: [0-9]{14,16} which checks that the user provided only numbers and that they entered at least 14 digits and at most 16 digits.
